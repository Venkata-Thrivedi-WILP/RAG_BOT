**Overview**
------------

Get authentication profile endpoint allows users to retrieve their authentication profile information. This endpoint uses a POST request to /AuthProfile/GetProfile and requires a UUID query parameter or as part of the payload.

Security: bearerAuth is required for this endpoint.

**Developer Notes**
------------------

### Required Fields

* `uuid`: The authentication profile UUID either passed in by method call or as part of the payload.

### Request Response Structure

* The response will be a JSON object with two properties: `Result` and `Error`.
* `Result` contains an object with nested properties:
	+ `Challenges`: An array of selected authentication mechanisms.
	+ `AdditionalData`: An optional data entity with additional data (default=null).
	+ `Uuid`: The profile UUID.
	+ `SingleChallengeMechanisms`: The selected single authentication mechanisms.
	+ `DurationInMinutes`: How many minutes the profile is valid.
	+ `Name`: The profile name.
* `Error` contains an object with error message text on failure, which may be null.

### Request/Response Schema

```json
{
  "uuid": "string_value"
}
```

```json
{
  "Result": {
    "Challenges": [
      "string_value"
    ],
    "AdditionalData": {},
    "Uuid": "string_value",
    "SingleChallengeMechanisms": "string_value",
    "DurationInMinutes": 0,
    "Name": "string_value"
  },
  "Error": {}
}
```

### Security Considerations

* bearerAuth is required for this endpoint.
