**Overview**
-----------

The Delete policy endpoint allows users to delete a specific policy block from their API. This endpoint is used in conjunction with the Policy Management tags.

To use this endpoint, you will need to send a POST request to `/Policy/DeletePolicyBlock` and provide a JSON payload containing the path of the policy block to be deleted. The `x-idap-anon` tag indicates that authentication is required for this endpoint.

**Developer Notes**
-----------------

### Required Parameters

* `path`: The policy block to delete (required)

### Request/Response Structure

* Request Body:
	+ `Content-Type: application/json`
	+ Schema Properties:
		- `path` (string): The policy block to delete
		- `Required fields`: `path`
* Response Body:
	+ `Result` (boolean): Did the policy block deletion succeed?
	+ `Error` (object): Error message text on failure, may be null

### Security Considerations

* Bearer authentication is required for this endpoint.
