# Overview  
The **Delete Policy** endpoint allows users to remove a specific policy block from the system. This is accomplished through a POST request to the path `/Policy/DeletePolicyBlock`. To access this endpoint, users must provide a valid bearer token for authentication.

# Developer Notes  
- **Required Parameters:**  
  - `path` (string): The policy block to delete. This field is mandatory in the request body.  
- **Request Structure:**  
  - Content Type: `application/json`  
  - Example Request JSON:  
    ```json
    {
      "path": "string_value"
    }
    ```
- **Response Structure:**  
  - Status Code: `200`  
  - Response Content Type: `*/*`  
  - Response Body Properties:  
    - `Result` (boolean): Indicates whether the policy block deletion was successful.  
    - `Error` (object): Contains error message text on failure, may be null.  
  - Example Response JSON:  
    ```json
    {
      "Result": false,
      "Error": {}
    }
    ```
- **Security Considerations:**  
  - This endpoint requires bearer token authentication.

# Detailed Explanation of Available Data, Request and Response Parameters  
## Request Parameters  
- **path**  
  - **Type:** string  
  - **Description:** The specific policy block that needs to be deleted.  
  - **Constraints:** This field is required.  
  - **Example Value:** `"policy/block/path"`

## Response Parameters  
- **Result**  
  - **Type:** boolean  
  - **Description:** Indicates if the deletion of the policy block was successful.  
  - **Example Value:** `true` or `false`  

- **Error**  
  - **Type:** object  
  - **Description:** Contains error message text if the deletion fails; may be null if there is no error.  
  - **Example Value:** `{}` or `{"message": "Policy block not found"}`
