# Overview
The **Get Policy Links** endpoint allows users to retrieve policy links associated with a specific policy set. This is done via a **POST** request to the path **/Policy/GetNicePlinks**. To access this endpoint, a valid **RevStamp** is required for both requesting and setting policy links. The response will return a dictionary containing the policy link details, with the **RevStamp** included as the last entry. Security is managed through **bearer authentication**.

# Developer Notes
- **Required Parameters**: 
  - RevStamp (must be included in the request)
- **Request Structure**: 
  - The request must be a POST to the endpoint `/Policy/GetNicePlinks`.
- **Response Structure**: 
  - The response will include a **Result** array with policy link details and an **Error** object if applicable.
- **Error Handling**: 
  - The **Error** object may contain error message text on failure, which can be null.
- **Security Considerations**: 
  - Bearer authentication is required to access this endpoint.

# Detailed Explanation of Available Data, Request and Response Parameters
### Request Parameters
- **RevStamp**: 
  - Type: String
  - Description: A unique token required for both requesting and setting policy links.

### Response Parameters
- **Result** (array): 
  - Description: The list of policy links in UI Query Response format.
  - Array items:
    - **Description** (string): The policy link description.
      - Example: `"User Data Policy"`
    - **I18NDescriptionTag** (string): Localized version of the description.
      - Example: `"user_data_policy"`
    - **PolicySet** (string): The policy link policy set.
      - Example: `"default_policy_set"`
    - **LinkType** (string): The policy link type.
      - Example: `"internal"`
    - **Params** (array): The policy link parameters.
      - Array items:
        - Example: `["param1", "param2"]`
    - **Filters** (array): The policy link filters.
      - Array items:
        - Example: `["filter1", "filter2"]`
    - **AllowedPolicies** (array): The policy link allowed policies.
      - Array items:
        - Example: `["policy1", "policy2"]`
    - **EnableCompliant** (boolean): Indicates if the policy link is enabled for compliance.
      - Example: `true`
- **Error** (object): 
  - Description: Error message text on failure, may be null.
