**Overview**
-----------

The `/Policy/GetNicePlinks` endpoint allows users to retrieve a list of policy links in UI Query Response format. This endpoint requires authentication using the `bearerAuth` security mechanism.

To use this endpoint, send a POST request to `/Policy/GetNicePlinks`. The request body should contain any required parameters.

**Developer Notes**
-------------------

### Required Parameters

* None specified in the raw documentation, but `RevStamp` is required.

### Request/Response Structure

* The request body should contain any required parameters.
* The response body is a dictionary with two properties: `Result` and `Error`.
* `Result` contains an array of policy links in UI Query Response format.
* `Error` may be null or contain error message text on failure.

### Error Handling

* If the request fails, the response will contain an `Error` object with error message text.

### Security Considerations

* This endpoint requires authentication using the `bearerAuth` security mechanism.

**Raw Endpoint Documentation (Formatted)**
-----------------------------------------

```markdown
ENDPOINT: Gets policy links.
PATH: /Policy/GetNicePlinks
METHOD: POST
TAGS: Policy Management
DESCRIPTION: RevStamp is required for both requesting and setting policy links. Return is a dictionary(plink name, plink fields as below) with RevStamp as the last entry.

METADATA:
  * x-idap-anon: False

RESPONSES:
  Status Code: 200
  Description: API-Result
  Content Type: */*
  Response Schema: PolicyGetNicePlinks
  Response Body Properties:
    * Result (array): The list of policy links in UI Query Response format.
      Array items:
        * Description (string): The policy link description.
        * I18NDescriptionTag (string): Localized version of the description.
        * PolicySet (string): The policy link policy set.
        * LinkType (string): The policy link type.
        * Params (array): The policy link parameters.
          Array items:
        * Filters (array): The policy link filters.
          Array items:
        * AllowedPolicies (array): The policy link allowed policies.
          Array items:
        * EnableCompliant (boolean): The policy link enable compliant.
    * Error (object): Error message text on failure, may be null
