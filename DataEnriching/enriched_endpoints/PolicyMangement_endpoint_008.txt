# Overview  
The **Get Policy Block** endpoint allows users to retrieve a specific policy block by name. This is useful for managing and understanding the configurations related to policy management within an application. The endpoint uses the **POST** HTTP method and can be accessed at the path **/Policy/GetPolicyBlock**. Security is enforced through bearer token authentication, ensuring that only authorized users can access the policy data.

# Developer Notes  
- **Required Parameters:**  
  - `name` (query, Required): The name of the policy block to retrieve.  
- **Request Structure:**  
  - The request must be made using the POST method with the required query parameter.  
- **Response Structure:**  
  - On success, the response will return a status code of 200 with the policy block details.  
  - If an error occurs, an error object will be returned.  
- **Security Considerations:**  
  - The endpoint requires bearer token authentication. Ensure that the token is included in the request headers.  

# Detailed Explanation of Available Data, Request and Response Parameters  
### Request Parameters  
- **name**  
  - **Type:** string  
  - **Description:** The name of the policy block to get.  
  - **Example Value:** `"examplePolicyBlock"`  

### Response Parameters  
- **Result** (object)  
  - **Description:** The policy block details.  
  - **Nested Properties:**  
    - **RiskAnalysisLevels** (object)  
      - **Description:** The risk analysis mapping.  
    - **AuthProfiles** (array)  
      - **Description:** A list of authentication profiles.  
      - **Items:** Objects representing each authentication profile.  
    - **PolicyModifiers** (array)  
      - **Description:** The policy modifiers.  
      - **Items:** Strings representing each policy modifier.  
    - **RevStamp** (string)  
      - **Description:** The policy revision stamp.  
      - **Example Value:** `"2023-10-01T12:00:00Z"`  
    - **RadiusClientList** (array)  
      - **Description:** The list of radius clients.  
      - **Items:** Objects representing each radius client.  
- **Error** (object)  
  - **Description:** Contains error details if the request fails.  
  - **Properties:**  
    - **description** (string): A description of the error.  
    - **type** (string): The type of the error.  
    - **properties** (object): Details about the properties of the error.
