**Overview**
-----------

The Get policy block endpoint allows users to retrieve a specific policy block from the system. This endpoint is used in policy management and requires authentication using the bearer token.

To use this endpoint, send a POST request to `/Policy/GetPolicyBlock` with the `name` parameter set to the ID of the policy block you want to retrieve.

**Security:** The endpoint uses bearer authentication, which means you need to provide an authorization token in the `Authorization` header of your request. This token is used to verify the identity of the user making the request.

**Developer Notes**
-------------------

### Required Parameters

* `name`: The ID of the policy block to retrieve (query parameter, required)

### Request/Response Structure

The request body should contain the `name` parameter. The response will be in JSON format and will include a `Result` object with properties such as `RiskAnalysisLevels`, `AuthProfiles`, `PolicyModifiers`, `RevStamp`, and `RadiusClientList`. If an error occurs, the response will include an `Error` object.

### Error Handling

The endpoint returns a 200 status code if successful. If an error occurs, it will return a 500 status code with an error message in the response body.

### Security Considerations

This endpoint uses bearer authentication, which means you need to provide an authorization token in the `Authorization` header of your request. Make sure to handle this token securely and do not store it in plain text.
