# Overview  
The **Delete a tag** endpoint allows users to remove a specified tag from the application. This operation is performed using the **POST** method at the path **/UPRest/DeleteTag**. To access this endpoint, users must authenticate using bearer token security.

# Developer Notes  
- **Required Parameters:**  
  - `tagname` (string): The name of the tag to be deleted.  
- **Request Structure:**  
  - Content Type: `application/json`  
  - Example Request JSON:  
    ```json
    {
      "tagname": "string_value"
    }
    ```  
- **Response Structure:**  
  - Status Code: `200`  
  - Response Body Properties:  
    - `Result` (object): Indicates whether the operation was successful.  
    - `Error` (object): Contains error message text on failure; may be null.  
  - Example Response JSON:  
    ```json
    {
      "Result": {},
      "Error": {}
    }
    ```  
- **Error Handling:** Ensure to check the `Error` object in the response for any issues that may arise during the deletion process.  
- **Security Considerations:** This endpoint requires bearer token authentication.

# Detailed Explanation of Available Data, Request and Response Parameters  
### Request Parameters  
- **tagname**  
  - **Type:** string  
  - **Description:** The name of the tag that you want to delete.  
  - **Constraints:** This field is required.  
  - **Example Value:** `"exampleTag"`  

### Response Parameters  
- **Result**  
  - **Type:** object  
  - **Description:** Indicates whether the delete operation was successful.  
  - **Example Value:** `{}` (empty object for success)  

- **Error**  
  - **Type:** object  
  - **Description:** Contains the error message text if the operation fails; may be null.  
  - **Example Value:** `{"message": "Tag not found"}`
