# Overview
This endpoint retrieves a list of details for recently uploaded files. It uses the HTTP POST method and is accessed at the path `/UPRest/GetRecentImportedAccountsFile`. To use this endpoint, you must provide a valid bearer authentication token, as it requires security credentials.

# Developer Notes
- **Required Parameters:**
  - `fileCount` (query, Required): Specifies the number of recent files to retrieve.
  
- **Request Structure:**
  - The request must be made via POST to the specified endpoint with the required query parameter.

- **Response Structure:**
  - **Status Code:** 200
  - **Content Type:** `*/*`
  - **Response Schema:** `UPRestGetRecentImportedAccountsFile`
  - **Response Body Properties:**
    - `Result` (object): Contains a list of details for recently uploaded files.
    - `Error` (object): Contains an error message text on failure; may be null.

- **Error Handling:**
  - The API will return an error object if the request fails, providing insight into the issue.

- **Security Considerations:**
  - This endpoint requires bearer authentication.

# Detailed Explanation of Available Data, Request and Response Parameters
### Request Parameters
- **fileCount**
  - **Type:** Integer
  - **Description:** The number of recent files to retrieve.
  - **Constraints:** Required parameter; must be a positive integer.
  - **Example Value:** `5`

### Response Parameters
- **Result**
  - **Type:** Object
  - **Description:** A list of details for recently uploaded files.
  - **Example Value:** 
    ```json
    {
      "FileName": "example_file.txt",
      "UploadDate": "2023-10-01T12:00:00Z",
      "FileSize": 2048
    }

  - **Type:** Object
  - **Description:** Contains error message text on failure; may be null.

    ```

}
```
