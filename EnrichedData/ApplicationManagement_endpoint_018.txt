# Overview  
This API endpoint allows users to retrieve tags associated with a specific application. Tags help in organizing applications by type. The endpoint uses the POST HTTP method and is accessed at the path `/UPRest/GetTagsForApp`. Only system administrators and users with application management rights are authorized to invoke this API. For further details, refer to the guides section [here](https://identity-developer.cyberark.com/docs/manage-applications-for-usersnew#create-user-tags-for-an-application). The endpoint requires bearer token authentication for security.

# Developer Notes  
- **Required Parameters:**  
  - `appkey` (query, Required): The name or Unique ID of the application.  
- **Request Structure:**  
  - The request must include the `appkey` parameter in the query.  
- **Response Structure:**  
  - On success, the response will contain a `Result` object.  
  - On failure, the response will include an `Error` object with error messages.  
- **Error Handling:**  
  - If the `appkey` is not provided or invalid, the error message will be: `'Parameter 'appkey' must be specified.'`  
- **Security Considerations:**  
  - The endpoint requires bearer token authentication.

# Detailed Explanation of Available Data, Request and Response Parameters  
## Request Parameters  
- **appkey**  
  - **Type:** string  
  - **Description:** The name or Unique ID of the application. This can be fetched from the Admin Portal after adding an application. The RowKey can also be obtained using a RedRock query. Note: The RowKey can be either the name or App key of the application but cannot be the Application Id for OAuth or OIDC applications.  
  - **Example Value:** `myAppKey123`  

## Response Parameters  
- **Response Status Code:** 200  
  - **Description:** Indicates a successful API call.  
- **Response Schema:** UPRestGetTagsForApp  
- **Response Body Properties:**  
  - **Result (object):** The return object for a successful API call.  
  - **Error (object):** Contains error message text on failure.  
    - **Nested Properties:**  
      - **AppKey_NotProvided (string):** This error message is displayed when an invalid rowkey is provided.  
        - **Example Value:** `'Parameter 'appkey' must be specified.'`  

### Sample Response JSON  
```json
{
  "Result": {},
  "Error": {
    "AppKey_NotProvided": "string_value"
  }
}
```
