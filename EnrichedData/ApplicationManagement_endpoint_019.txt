**Overview**
-----------

The Rename a tag endpoint allows users to update the name of an existing tag. This endpoint is part of the Application Management tags and requires authentication using the Bearer token.

To use this endpoint, send a POST request to `/UPRest/RenameTag` with a JSON payload containing the new tag name and the current tag name. The `x-idap-anon` header should be set to `False`, indicating that an anonymous user is not requesting access.

**Developer Notes**
-------------------

### Required Fields

* `newTagname`: The new name of the tag (string)
* `tagname`: The current name of the tag (string)

### Request/Response Structure

* Request Body: JSON payload with `newTagname` and `tagname`
	+ Content Type: application/json
	+ Schema Properties:
		- `newTagname` (string): new tag name
		- `tagname` (string): tag name
* Response Body: UPRestCreateTagWithNoApp schema with `Result` and `Error` properties

### Error Handling

* If the operation fails, an error message will be returned in the `Error` property of the response body.

### Security Considerations

* Bearer authentication is required for this endpoint.
* The `x-idap-anon` header should be set to `False` to indicate that an anonymous user is not requesting access.
