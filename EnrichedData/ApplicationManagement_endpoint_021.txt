**Overview**
================

Update captured user application. This endpoint allows you to update an existing application in the system.

* HTTP Method: POST
* Path: `/UPRest/UpdateCapturedUserApplication`
* Security Requirement: Bearer Authentication

**Developer Notes**
-------------------

### Required Fields

* `appkey` (string): Application key
* `appName` (string): Application Name
* `appDescription` (string): Application Description
* `notes` (string): Application notes
* `appUrl` (string): Application Url

### Request/Response Structure

* Request Body: JSON object with required fields
	+ Sample Request JSON:
```json
{
  "appName": "string_value",
  "appkey": "string_value",
  "appDescription": "string_value",
  "notes": "string_value",
  "appUrl": "string_value"
}
```
* Response Body: JSON object with `Result` and optional `Error` properties
	+ Sample Response JSON:
```json
{
  "Result": {},
  "Error": {}
}
```

### Error Handling

* If the request body is missing required fields, a 400 Bad Request error will be returned.
* If the application key is invalid or missing, a 401 Unauthorized error will be returned.

### Security Considerations

* Bearer authentication is required for this endpoint.
