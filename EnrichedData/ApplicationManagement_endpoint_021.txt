# Overview  
The **Update Captured User Application** endpoint allows users to update the details of a captured application within the system. This is done through a POST request to the path `/UPRest/UpdateCapturedUserApplication`. The endpoint requires authentication via a bearer token, ensuring that only authorized users can make updates to application data.

# Developer Notes  
- **HTTP Method**: POST  
- **Path**: `/UPRest/UpdateCapturedUserApplication`  
- **Security**: Requires bearer authentication.  
- **Required Parameters**:  
  - `appkey` (string): This is the only required field for the request.  
- **Request Structure**: The request body must be in JSON format and include the following fields:  
  - `appName` (string): Application Name  
  - `appkey` (string): Application key (required)  
  - `appDescription` (string): Application Description  
  - `notes` (string): Application notes  
  - `appUrl` (string): Application URL  
- **Response Structure**: The response will return a status code of 200 with a JSON body containing:  
  - `Result` (object): Indicates whether the operation was successful.  
  - `Error` (object): Contains error message text on failure, may be null.  
- **Error Handling**: Ensure to check the `Error` field in the response for any issues.

# Detailed Explanation of Available Data, Request and Response Parameters  
## Request Parameters  
- **appName**  
  - **Type**: string  
  - **Description**: The name of the application.  
  - **Example**: `"My Application"`  

- **appkey**  
  - **Type**: string  
  - **Description**: The unique key for the application (required).  
  - **Example**: `"app_key_123"`  

- **appDescription**  
  - **Type**: string  
  - **Description**: A brief description of the application.  
  - **Example**: `"This application does XYZ."`  

- **notes**  
  - **Type**: string  
  - **Description**: Additional notes regarding the application.  
  - **Example**: `"This is a note about the application."`  

- **appUrl**  
  - **Type**: string  
  - **Description**: The URL where the application can be accessed.  
  - **Example**: `"https://www.example.com"`  

## Response Parameters  
- **Result**  
  - **Type**: object  
  - **Description**: Indicates whether the operation was successful.  
  - **Example**: `{}`  

- **Error**  
  - **Type**: object  
  - **Description**: Contains error message text on failure, may be null.  
  - **Example**: `{"message": "An error occurred."}`
