**Overview**
-----------

The `/UPRest/UpsertTagsForApp` endpoint allows users to add and update application tags for the current user. This endpoint is accessible via a POST request.

To use this endpoint, you will need to provide an authentication token in the `Authorization` header using bearerAuth. The request body should be in JSON format with the required fields: `appkey` and `tagnames`.

**Developer Notes**
-------------------

### Request Structure

* HTTP Method: POST
* Path: `/UPRest/UpsertTagsForApp`
* Content Type: `application/json`
* Required Fields:
	+ `appkey` (string): Application key
	+ `tagnames` (array): list of tag names
* Sample Request JSON:
```json
{
  "appkey": "string_value",
  "tagnames": [
    "string_value"
  ]
}
```

### Response Structure

* Status Code: 200
* Content Type: `*/*`
* Response Schema: `UPRestCreateTagWithNoApp`
* Sample Response JSON:
```json
{
  "Result": {},
  "Error": {}
}
```

### Security Considerations

* Bearer authentication required in the `Authorization` header.
