**Overview**
-----------

Get TOTP codes for a Secured Password
=====================================

This endpoint allows users to retrieve TOTP (Time-Based One-Time Password) codes for a secured password. The default value for the number of TOTP codes (`n`) is 3, but it can be configured.

To use this endpoint, you must provide the `sItemkey` of the secured item and optionally specify an `actionType`. The request body should contain the required fields in JSON format.

**Developer Notes**
------------------

### Required Fields

* `sItemkey` (string): Secured item's unique ID
* `actionType` (string): Optional. The action item for audit events includes the type of action (AutoFill, IBECopy, ContextMenuCopy). By default, it is set to 'null'.

### Request Body

The request body should contain the required fields in JSON format:
```json
{
  "sItemkey": "string_value",
  "actionType": "string_value"
}
```

### Response Structure

The response will be in the following format:
```json
{
  "Result": {
    "success": boolean,
    "Result": {
      "Period": integer,
      "TOTPCodes": {
        "1": string,
        "2": string,
        "3": string
      },
      "EpochTimeInSec": integer
    },
    "Message": string,
    "MessageID": string,
    "IsSoftError": boolean,
    "Exception": string,
    "ErrorID": string,
    "ErrorCode": string,
    "InnerExceptions": string
  },
  "Error": {
    "UnAuthorized": string
  }
}
```

### Security

This endpoint uses bearer authentication.
