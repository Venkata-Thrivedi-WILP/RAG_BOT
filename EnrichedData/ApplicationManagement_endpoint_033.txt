# Overview  
This endpoint allows users to add, update, or delete file attachments for a secured item within the application. It is accessed via a POST request to the path `/UPRest/UpRestSaveFilesForSecuredItem`. To utilize this endpoint, users must authenticate using bearer token authentication.

# Developer Notes  
- **Required Parameters:**  
  - The request body must be in JSON format and include the following required fields:
    - `File`: The file to be uploaded in form-data.
  
- **Request Structure:**  
  - Content Type: `application/json`
  - Schema Properties:
    - `Metadata`: An array holding file metadata.
      - Items:
        - `Name` (string): Filename.
        - `Id` (string): Unique file identifier.
        - `Hidden` (boolean): Enables or disables the file.
        - `Size` (number): File size.
    - `File` (object): The file is included in the form-data.

- **Response Structure:**  
  - Status Code: `200` indicates a successful operation.
  - Response Body Properties:
    - `Result` (object): Indicates whether the operation was successful.
    - `Error` (object): Contains error message text on failure, may be null.

- **Security Considerations:**  
  - Bearer token authentication is required to access this endpoint.

# Detailed Explanation of Available Data, Request and Response Parameters  

## Request Parameters  
- **Metadata** (array): Holds file metadata.
  - **Items:**
    - **Name** (string): The name of the file (e.g., `"document.pdf"`).
    - **Id** (string): A unique identifier for the file (e.g., `"file12345"`).
    - **Hidden** (boolean): Indicates whether the file is hidden (e.g., `false`).
    - **Size** (number): The size of the file in bytes (e.g., `2048`).

- **File** (object): The actual file to be uploaded, provided in the form-data.

### Sample Request JSON  
```json
{
  "Metadata": [
    {
      "Name": "string_value",
      "Id": "string_value",
      "Hidden": false,
      "Size": 0
    }
  ],
  "File": {}
}
```

## Response Parameters  
- **Result** (object): Indicates whether the operation was successful.
- **Error** (object): Contains error message text on failure, may be null.

### Sample Response JSON  
```json
{
  "Result": {},
  "Error": {}
}
```
