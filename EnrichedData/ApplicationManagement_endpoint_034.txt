# Overview
This endpoint allows users to retrieve a list of secured files attached to a specific Secured Item. It is accessed via a POST request to the path `/UPRest/GetSecureFiles`. To use this endpoint, users must provide a valid `sItemkey` parameter, which represents the unique identifier for the secured item. Additionally, the endpoint requires bearer token authentication for security purposes.

# Developer Notes
- **Required Parameters**:
  - `sItemkey` (query, Required): This is the unique identifier for the secured item.
  
- **Request Structure**:
  - Method: POST
  - Path: `/UPRest/GetSecureFiles`
  - Authentication: Bearer token required.

- **Response Structure**:
  - Status Code: 200
  - Content Type: `*/*`
  - Response Schema: `UPRestGetRecentImportedAccountsFile`
  
- **Response Body Properties**:
  - `Result` (object): Contains a list of details for recently uploaded files.
  - `Error` (object): Contains error message text on failure, may be null.

- **Error Handling**: The response will include an error message if the request fails.

# Detailed Explanation of Available Data, Request and Response Parameters

## Request Parameters
- **sItemkey**
  - **Type**: String
  - **Description**: The unique identifier for the secured item.
  - **Constraints**: Required for the request.

## Response Parameters
- **Response Body**
  - **Result** (object)
    - **Description**: A list of details for recently uploaded files.
    - **Example**: 
      ```json

        "UploadDate": "2023-10-01T12:00:00Z"
      }
      ```

    - **Description**: Contains error message text if the request fails, may be null.

      }
      ```
