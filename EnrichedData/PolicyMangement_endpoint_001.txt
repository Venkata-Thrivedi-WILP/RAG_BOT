# Overview  
The **Delete Authentication Profile** endpoint allows users to delete an existing authentication profile by providing its unique identifier (UUID). This endpoint uses the HTTP POST method and is accessed via the path `/AuthProfile/DeleteProfile`. To use this endpoint, users must authenticate with a bearer token.

# Developer Notes  
- **Required Parameters:**  
  - `uuid` (query, Required): The UUID of the authentication profile to be deleted.  
- **Request Structure:**  
  - Content Type: `application/json`  
  - Required fields in the request body:  
    - `uuid` (string): The authentication profile UUID.  
- **Response Structure:**  
  - Status Code: `200`  
  - Response Body Properties:  
    - `Result` (boolean): Indicates success or failure of the delete operation.  
    - `Error` (object): Contains error message text on failure, may be null.  
- **Error Handling:** Ensure to check the `Result` field in the response to determine if the operation was successful.  
- **Security Considerations:** This endpoint requires bearer token authentication.

# Detailed Explanation of Available Data, Request and Response Parameters  

## Request Parameters  
- **uuid**  
  - **Type:** string  
  - **Description:** The unique identifier for the authentication profile to be deleted.  
  - **Constraints:** Required field, must be a valid UUID format.  
  - **Example:** `"123e4567-e89b-12d3-a456-426614174000"`  

## Request Body  
```json
{
  "uuid": "string_value"
}
```

## Response Parameters  
- **Result**  
  - **Type:** boolean  
  - **Description:** Indicates whether the deletion was successful.  
  - **Example:** `false`  

- **Error**  
  - **Type:** object  
  - **Description:** Contains error message text on failure, may be null.  
  - **Example:** `{}`  

## Sample Response JSON  
```json
{
  "Result": false,
  "Error": {}
}
```
