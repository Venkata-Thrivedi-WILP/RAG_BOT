**Overview**
-----------

The `/AuthProfile/SaveProfile` endpoint allows users to save their authentication profile. This endpoint is used for updating an existing profile or creating a new one if no UUID is provided.

To use this endpoint, you must include a JSON payload in the request body with the required `settings` field. The HTTP method is POST and the path is `/AuthProfile/SaveProfile`.

Security authentication is required using bearer tokens.

**Developer Notes**
------------------

### Required Fields

* `settings`: The authentication profile.

### Request/Response Structure

* Request Body:
	+ `Content Type`: application/json
	+ `Schema Properties`:
		- `settings`: The authentication profile.
			Nested properties:
				- `Challenges`: Selected authentication mechanisms.
					Array items:
				- `AdditionalData`: DataEntity with additional data (default=null).
				- `Uuid`: Profile UUID.
				- `SingleChallengeMechanisms`: Selected single authentication mechanisms.
				- `DurationInMinutes`: How many minutes the profile is valid.
				- `Name`: Profile name.
			Required fields: `settings`
* Response Body:
	+ `Result`: An authentication profile.
		Nested properties:
			- `Challenges`: Selected authentication mechanisms.
				Array items:
			- `AdditionalData`: DataEntity with additional data (default=null).
			- `Uuid`: Profile UUID.
			- `SingleChallengeMechanisms`: Selected single authentication mechanisms.
			- `DurationInMinutes`: How many minutes the profile is valid.
			- `Name`: Profile name.
	+ `Error`: Error message text on failure, may be null

### Security Considerations

* Bearer authentication is required for this endpoint.
