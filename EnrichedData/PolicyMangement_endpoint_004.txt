# Overview  
The **Delete Policy** endpoint allows users to remove a specific policy block from the system. It is accessed via a POST request to the path **/Policy/DeletePolicyBlock**. To use this endpoint, users must authenticate using bearer token security.

# Developer Notes  
- **Required Parameters:**  
  - `path` (string): The policy block to delete.  
- **Request Structure:**  
  - Content Type: `application/json`  
  - Example Request JSON:  
    ```json
    {
      "path": "string_value"
    }
    ```
- **Response Structure:**  
  - Status Code: `200`  
  - Response Body Properties:  
    - `Result` (boolean): Indicates if the policy block deletion succeeded.  
    - `Error` (object): Contains error message text on failure, may be null.  
  - Example Response JSON:  
    ```json
    {
      "Result": false,
      "Error": {}
    }
    ```
- **Security Considerations:**  
  - This endpoint requires bearer token authentication.

# Detailed Explanation of Available Data, Request and Response Parameters  
### Request Parameters  
- **path**  
  - **Type:** string  
  - **Description:** The specific policy block that needs to be deleted.  
  - **Constraints:** This field is required.  
  - **Example Value:** `"policy/block/123"`  

### Response Parameters  
- **Result**  
  - **Type:** boolean  
  - **Description:** Indicates whether the deletion of the policy block was successful.  
  - **Example Value:** `true` or `false`  

- **Error**  
  - **Type:** object  
  - **Description:** Contains error message text if the deletion fails; otherwise, it may be null.  
  - **Example Value:** `{ "message": "Policy block not found." }` or `{}`
