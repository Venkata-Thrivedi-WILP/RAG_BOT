**Overview**
==========

Delete policy endpoint allows users to delete a specific policy block from their system. This endpoint is used in policy management and requires authentication via bearer token.

To use this endpoint, you need to send a POST request to `/Policy/DeletePolicyBlock` with the policy block path as a JSON payload. The endpoint uses bearer authentication for security purposes.

**Developer Notes**
================

### Required Fields

* `path`: The policy block to delete. (string)

### Request/Response Structure

* Request Body: JSON payload with `path` field.
* Response Body: JSON object with `Result` and `Error` fields.

### Error Handling

* If the policy block does not exist, an error message will be returned in the response body.
* If the deletion fails for any reason, a boolean value of `false` will be returned in the `Result` field, along with an error message in the `Error` field.

### Security Considerations

* Bearer authentication is required for this endpoint. A valid bearer token must be included in the request headers.
