# Overview  
The **Get Authentication Assurance Level** endpoint allows users to retrieve the authentication assurance levels (AAL) associated with a specific profile. This is done using a POST request to the path **/AuthProfile/GetProfileMFAScoring**. The endpoint requires bearer token authentication for security, ensuring that only authorized users can access the information.

# Developer Notes  
- **Required Parameters:**  
  - `challenges` (query, Required): Profile factors that need to be provided in the request.  

- **Request Structure:**  
  - The request should be sent as a POST with the required query parameter.

- **Response Structure:**  
  - **Status Code:** 200  
  - **Response Body Properties:**  
    - `Result` (object): Contains the maximum and minimum AAL in case of success or null in case of failure.  
    - `Error` (object): Provides an error message text on failure, may be null.  

- **Error Handling:**  
  - The API will return an error object if the request fails, which should be checked in the response.

- **Security Considerations:**  
  - This endpoint requires bearer token authentication.

# Detailed Explanation of Available Data, Request and Response Parameters  
## Request Parameters  
- **challenges**  
  - **Type:** Query  
  - **Description:** Profile factors that are necessary for determining the authentication assurance level.  
  - **Constraints:** This parameter is required.  
  - **Example Value:** `{"factor1": "value1", "factor2": "value2"}`  

## Response Parameters  
- **Result**  
  - **Type:** Object  
  - **Description:** Returns the maximum and minimum AAL on success or null on failure.  
  - **Example Value:**  
    ```json
    {
      "maxAAL": 3,
      "minAAL": 1
    }
    ```

- **Error**  
  - **Type:** Object  
  - **Description:** Contains an error message if the request fails, may be null.  
  - **Example Value:**  
    ```json
    {
      "message": "Invalid challenges parameter"
    }
    ```
