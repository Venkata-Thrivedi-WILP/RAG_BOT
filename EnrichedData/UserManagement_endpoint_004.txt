**Overview**
==========

Invite Cloud Users endpoint allows system administrators or users with user management permissions to invite new users or groups to their organization. This endpoint uses a POST request to /UserMgmt/InviteUsers.

To use this endpoint, you must authenticate with the API using bearer authentication. You can find more information on authentication in the Security section below.

**Developer Notes**
==================

### Required Parameters

* `Entities`: An array of dictionaries containing information about the users or groups to invite. Each dictionary must include:
	+ `Type` (string): The type of entity being invited (User, Group, Role)
	+ `Guid` (string): The UUID of the entity being invited
	+ `Name` (string): The name of the entity being invited (only applicable to User entities)
* `EmailInvite`: A boolean indicating whether to send email invitations to invited entities (default=true)

### Request/Response Structure

The request body is expected in JSON format and must contain the required fields. The response will be a JSON object with a `Result` property containing the API result and an optional `Error` property containing error messages.

### Error Handling

* If the request fails, the response will contain an error message in the `Error` property.
* If the request is successful, the response will contain an empty `Error` property.

### Security Considerations

* Bearer authentication is required for this endpoint. See the Security section above for more information on authentication.
