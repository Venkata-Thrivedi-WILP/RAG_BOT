# Overview
The **Invite Cloud Users** endpoint allows system administrators or users with user management permissions to invite new users or groups to the cloud system. This is accomplished through a POST request to the path `/UserMgmt/InviteUsers`. The endpoint requires bearer token authentication for security.

# Developer Notes
- **Required Parameters**: 
  - `Entities`: Must include user or group details.
  - `EmailInvite`: Must be set to true or false.
- **Request Structure**: The request body must be in JSON format and include the required fields.
- **Response Structure**: The response will contain a result object and may include an error message.
- **Error Handling**: Check the `Error` field in the response for any issues.
- **Security Considerations**: Ensure that the request includes a valid bearer token for authentication.

# Detailed Explanation of Available Data, Request and Response Parameters
## Request Parameters
- **SmsInvite** (boolean): 
  - Description: Indicates whether to send SMS invitations to invited entities.
  - Default: `false`
  
- **Entities** (array): 
  - Description: List of users or groups to invite. Each entry must include:
    - **Type** (string): Type of entity (User, Group, Role).
    - **Guid** (string): UUID of the entity.
    - **Name** (string): Name of the entity (only applicable to User entities).
  
- **Role** (string): 
  - Description: Role name to which each invited user will be added (default is `null`).
  
- **GroupInvite** (boolean): 
  - Description: If true, sends a single email invitation to the group rather than individual emails (default: `true`).
  
- **EmailInvite** (boolean): 
  - Description: Indicates whether to send email invitations to invited entities.
  - Required: Yes

### Sample Request JSON
```json
{
  "SmsInvite": false,
  "Entities": [
    {
      "Type": "User",
      "Guid": "123e4567-e89b-12d3-a456-426614174000",
      "Name": "John Doe"
    }
  ],
  "Role": "Admin",
  "GroupInvite": false,
  "EmailInvite": true
}
```

## Response Parameters
- **Status Code**: `200`
- **Description**: API-Result
- **Content Type**: `*/*`
  
### Response Body Properties
- **Result** (object): 
  - Description: Standard result object.
  
- **Error** (object): 
  - Description: Contains error message text on failure; may be `null`.

### Sample Response JSON
```json
{
  "Result": {},
  "Error": null
}
```

**ENDPOINT**: Invite Cloud Users  
**PATH**: /UserMgmt/InviteUsers  
**METHOD**: POST  
**TAGS**: Send User Invites  
**DESCRIPTION**: You must be a system administrator or have user management permissions.  
**METADATA**:  
  * x-idap-anon: False  
  * x-codegen-request-body-name: payload  
**REQUEST BODY**: Required  
  Content Type: application/json  
  Schema Properties:  
    * SmsInvite (boolean): Send SMS invitations to invited entities if possible (default=false)  
    * Entities (array): Users or groups to invite; each entry is a dictionary including 'Type' (User, Group, Role), 'Guid' (entity UUID), and 'Name' (entity name, only applicable to User entities)  
      Array items:  
    * Role (string): Add each invited user to this role name if specified (default=null)  
    * GroupInvite (boolean): For invited groups, send a single email invitation to the group rather than to each user within the group (default=true)  
    * EmailInvite (boolean): Send email invitations to invited entities if possible (default=true)  
    **Required fields**: Entities, EmailInvite  
  **Sample Request JSON**:  
  ```json
  {
    "SmsInvite": false,
    "Entities": [
      {}
    ],
    "Role": "string_value",
    "GroupInvite": false,
    "EmailInvite": false
  }
  ```  
**RESPONSES**:  
  Status Code: 200  
  Description: API-Result  
  Content Type: */*  
  Response Schema: UserMgmtInviteUsers  
  Response Body Properties:  
    * Result (object): Standard result  
    * Error (object): Error message text on failure, may be null  
  **Sample Response JSON**:  
  ```json
  {
    "Result": {},
    "Error": {}
  }
  ```
