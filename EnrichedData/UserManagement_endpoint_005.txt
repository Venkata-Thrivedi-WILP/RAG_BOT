# Overview
The **Send Invitation Email** endpoint allows system administrators to send login invitations to users by providing their UUIDs. This endpoint uses the HTTP POST method and is accessed at the path `/UserMgmt/SendLoginEmails`. To use this endpoint, the requester must have appropriate security credentials, specifically a bearer token for authentication.

# Developer Notes
- **Required Parameters:**
  - Request Body: Must include the `ID` field.
- **Request Structure:**
  - Content Type: `application/json`
  - Required Fields: 
    - `ID` (array): List of user UUIDs to send invitations to.
- **Response Structure:**
  - Status Code: 200
  - Response Content Type: `*/*`
  - Response Schema: `UserMgmtInviteUsers`
  - Response Body Properties:
    - `Result` (object): Standard result.
    - `Error` (object): Error message text on failure, may be null.
- **Error Handling:** Ensure to handle potential error messages in the response.
- **Security Considerations:** Requires bearer authentication.

# Detailed Explanation of Available Data, Request and Response Parameters
## Request Parameters
- **ID (array)**: 
  - **Type:** Array of strings
  - **Description:** List of user UUIDs to whom invitations will be sent.
  - **Constraints:** Must contain at least one UUID.
  - **Example:**
    ```json
    {
      "ID": [
        "123e4567-e89b-12d3-a456-426614174000"
      ]
    }
    ```

## Response Parameters
- **Result (object)**: 
  - **Description:** Standard result object returned on successful request.
- **Error (object)**: 
  - **Description:** Contains error message text if the request fails; may be null.
  - **Example:**
    ```json
    {
      "Result": {},
      "Error": {}
    }
    ```
