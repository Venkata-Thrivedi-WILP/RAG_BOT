# Overview  
The **Send Invitation SMS** endpoint allows system administrators or users with user management permissions to send SMS invitations to users. This is accomplished through a POST request to the path `/UserMgmt/SendSmsInvite`. Access to this endpoint requires bearer token authentication.

# Developer Notes  
- **Required Parameters:**  
  - `ID` (query, Required): User UUID to send the invitation to.  
- **Request Structure:**  
  - The request must be a POST to `/UserMgmt/SendSmsInvite` with the required query parameter.  
- **Response Structure:**  
  - Status Code: 200  
  - Content Type: `*/*`  
  - Response Schema: `UserMgmtInviteUsers`  
  - Response Body Properties:  
    - `Result` (object): Contains standard result information.  
    - `Error` (object): Contains error message text on failure, may be null.  
- **Error Handling:**  
  - Check the `Error` object in the response for details if the request fails.  
- **Security Considerations:**  
  - This endpoint requires bearer token authentication.

# Detailed Explanation of Available Data, Request and Response Parameters  
### Request Parameters  
- **ID**  
  - **Type:** Query  
  - **Description:** The UUID of the user to whom the invitation will be sent.  
  - **Constraints:** Required field.  
  - **Example Value:** `123e4567-e89b-12d3-a456-426614174000`  

### Response Parameters  
- **Result**  
  - **Type:** Object  
  - **Description:** Standard result information returned upon successful processing of the request.  
- **Error**  
  - **Type:** Object  
  - **Description:** Contains error message text if the request fails; may be null.  

### Sample Response JSON  
```json
{
  "Result": {},
  "Error": {}
}
```
