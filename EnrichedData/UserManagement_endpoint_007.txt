# Overview
This API endpoint allows you to fetch the details of a specific existing user in the cloud directory. It is a POST request made to the path `/CDirectoryService/GetUser`. Only system administrators, users with user management rights, or the user themselves can invoke this API. Standard users can only retrieve their own details, while administrators can access details of any user. The request requires the unique ID of the user as a parameter in the request body. The response includes the user's ID, name, and various other details.

**Security Requirements:**  
This endpoint requires bearer token authentication.

# Developer Notes
- **Required Parameters:** The request body must include the following required field:
  - `ID` (string): The unique ID of the cloud user to fetch. Example: `abc-e2e-123`
  
- **Request Structure:**
  - Content Type: `application/json`
  
- **Response Structure:**
  - Status Code: `200`
  - Content Type: `*/*`
  
- **Error Handling:** 
  - The API returns various fields in the response to indicate success or failure, including `ErrorID`, `ErrorCode`, and `Message`.
  - If a user without administrative/user management permission tries to fetch details of another user, an error message will be returned indicating unauthorized access.

# Detailed Explanation of Available Data, Request and Response Parameters
## Request Parameters
- **ID** (string): 
  - Description: The unique identifier for the user whose details are being fetched.
  - Example: `"ID": "abc-e2e-123"`

## Response Parameters
- **Result** (object): 
  - Description: Contains the details of the user.
  - Nested Properties:
    - **ErrorID** (string): Error ID if any error is received from the API. Default is `null`.
    - **ErrorCode** (string): Error code if any error is received from the API. Default is `null`.
    - **Exception** (string): Exception message if an exception is received from the API. Default is `null`.
    - **MessageID** (string): Message ID for failure response. Default is `null`.
    - **Result** (object): 
      - Description: Includes total count and details of the user.
      - Nested Properties:
        - **Alias** (string): Suffix for the tenant. Example: `@acme.inc`
        - **Version** (string): Version of the data being received. Example: `1.0`
        - **DisplayName** (string): Display name of the user. Default is `null`.
        - **CreateDate** (string): Date/time user was created.
        - **LastPasswordChangeDate** (string): Date/time user last changed their password.
        - **MobileNumber** (string): User's mobile number. Default is `null`.
        - **InEverybodyRole** (boolean): Indicates if the user is in the Everybody role.
        - **LastModifiedDate** (string): Date/time user was last modified.
        - **State** (string): User state (e.g., locked, enabled, expired).
        - **Mail** (string): Email address of the user. Example: `'mark@email.com'`
        - **Name** (string): Username of the user. Example: `'mark@acme.inc'`
    - **InnerExceptions** (string): Inner exceptions if any received from the API. Default is `null`.
    - **Message** (string): Error message for failure response. Default is `null`.
    - **success** (boolean): Indicates if the API call was successful (true or false).
    - **IsSoftError** (boolean): If true, indicates this is not an exception but a UI warning popup.
  
- **Error** (object): 
  - Description: Error message text on failure.
  - Nested Properties:
    - **UnauthorizedUser** (string): Message indicating unauthorized access. Example: `"You are not authorized to perform this operation. Please contact your IT helpdesk"`
