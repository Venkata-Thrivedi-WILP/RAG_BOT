# Overview
The **Update Base Profile** endpoint allows you to update the information for a specified user within the system. This endpoint is accessed via a **POST** request at the path **/CDirectoryService/ChangeUser**. It is intended for cloud users and can only be invoked by system administrators, users with user management rights, or the user themselves. Additionally, if both 'ServiceUser' and 'InEverybodyRole' properties are provided, specifying 'ServiceUser' will take precedence. This endpoint requires bearer token authentication for security.

# Detailed Explanation of Available Data, Request and Response Parameters
## Request Parameters
- **CmaRedirectedUserUuid** (string): Idaptive mobile authenticator redirected user unique id.
- **HomeNumber** (string): User's home number.
- **useAlternateMfaAccount** (boolean): Indicates if the user has an alternate MFA account.
- **OrgPath** (string): The organization name to which the user is added.
- **AccountExp** (string): Account expiry date/time.
- **ReportsTo** (string): Unique Id of the user to whom the user reports.
- **DisplayName** (string): User's display name.
- **PasswordNeverExpire** (boolean): Indicates if the user's password expires.
- **ID** (string, Required): Unique Id of the cloud user to change. Example: `a3e5bab3-9104-49cb-9d3a-e5da7gcef3a3`
- **OfficeNumber** (string): User's office number.
- **ForcePasswordChangeNext** (boolean): Indicates if the user should change password on first login.
- **MobileNumber** (string): User's mobile number.
- **InEverybodyRole** (boolean): Indicates if the user is in the Everybody role.
- **ServiceUser** (boolean): Indicates if the user is a service user.
- **VerifyEmail** (boolean): Enforces user to verify the registered email address.
- **PreferredCulture** (string): User's preferred culture.
- **Mail** (string): User's email address.
- **Description** (string): User's description.
- **Name** (string): Username of the user. Format can be 'username@suffix' if suffix is required, else 'username'.

## Response Parameters
- **Status Code**: 200
- **Response Schema**: CDirectoryServiceChangeUser
- **Response Body Properties**:
  - **Result** (object): Return object for a successful API call.
    - **ErrorID** (string): Error ID if any error is received from the API. Default is 'null'.
    - **ErrorCode** (string): Error code if any error is received from the API. Default is 'null'.
    - **Exception** (string): Exception message if any exception is received from the API. Default is 'null'.
    - **MessageID** (string): Message Id for failure response. Default is 'null'.
    - **Result** (object): The result for the API call.
    - **InnerExceptions** (string): Inner Exceptions if any received from the API. Default is 'null'.
    - **Message** (string): Error message for failure response. Default is 'null'.
    - **success** (boolean): True for a successful API call.
    - **IsSoftError** (boolean): Indicates if this is not an actual exception but a UI warning popup.
  - **Error** (object): Error message text on failure.
    - **InvalidId** (string): Error message when the unique user Id is incorrect. Default message: "You are not authorized to perform this operation. Please contact your IT helpdesk."

### Sample Response JSON
```json
{
  "Result": {
    "ErrorID": "string_value",

    "Exception": "string_value",
    "MessageID": "string_value",

    "Message": "string_value",
    "success": false,
    "IsSoftError": false
  },
  "Error": {
    "InvalidId": "string_value"
  }
}
```
