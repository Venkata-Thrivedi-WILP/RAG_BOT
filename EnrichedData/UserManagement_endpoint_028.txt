# Overview
The **Disable/enable a user** endpoint allows you to lock or unlock a user account within the system. This functionality is primarily intended for system administrators, users with user management rights, or the user themselves. The endpoint uses the HTTP POST method and is accessed via the path `/UserMgmt/SetCloudLock`. To use this endpoint, a bearer token for authentication is required.

# Developer Notes
- **Required Parameters:**
  - `user` (query): The unique ID of the user. This ID is generated when the user is created.
  - `lockUser` (query): A boolean parameter that specifies whether the user account should be locked (`true`) or not (`false`). The default value is `false`.

- **Request Structure:**
  - The request must include the required parameters in the query string.

- **Response Structure:**
  - The response will include a status code of `200` for a successful API call.
  - The response body will contain a `Result` object and an `Error` object in case of failure.

- **Error Handling:**
  - The API may return an error message if the user ID is invalid or if there are other issues.
  - The `Error` object will provide details about the error encountered.

- **Security Considerations:**
  - This endpoint requires bearer authentication.

# Detailed Explanation of Available Data, Request and Response Parameters

## Request Parameters
- **user** (query, Required)
  - **Type:** String
  - **Description:** The unique ID of the user.
  - **Example:** `"12345"`

- **lockUser** (query, Required)
  - **Type:** Boolean
  - **Description:** Indicates whether to lock the user account. Default is `false`.
  - **Example:** `true`

## Response Parameters
- **Result** (object)
  - **Description:** Return object for a successful API call.
  - **Properties:**
    - **ErrorID** (string): Error ID if any error is received from the API. Default is `null`.
    - **ErrorCode** (string): Error code if any error is received from the API. Default is `null`.
    - **Exception** (string): Exception message if any exception is received from the API. Default is `null`.
    - **MessageID** (string): Message ID for failure response. Default is `null`.
    - **Result** (string): The result will be null.
    - **InnerExceptions** (string): Inner exceptions.
    - **Message** (string): Error message for failure response. Default is `null`.
    - **success** (boolean): Indicates if the operation was successful (`true` or `false`).
    - **IsSoftError** (boolean): Indicates if this is a soft error (UI warning) if true.

- **Error** (object)
  - **Description:** Error message text on failure.
  - **Properties:**
    - **InvalidUserId** (string): Message indicating that the user with the given ID was not found.
