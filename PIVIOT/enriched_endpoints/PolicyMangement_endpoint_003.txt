**Overview**
-----------

The `/AuthProfile/SaveProfile` endpoint allows users to save their authentication profile. This endpoint is used for updating an existing profile if it contains a UUID, or creating a new profile.

To use this endpoint, you will need to send a POST request with the required `settings` object in the request body. The request body should be in JSON format and contain the following properties:

*   `Challenges`: An array of selected authentication mechanisms.
*   `AdditionalData`: DataEntity with additional data (default=null).
*   `Uuid`: Profile UUID.
*   `SingleChallengeMechanisms`: Selected single authentication mechanisms.
*   `DurationInMinutes`: How many minutes the profile is valid.
*   `Name`: Profile name.

The endpoint requires bearer authentication and returns a 200 status code if successful. The response body will contain an authentication profile object with the same properties as the request body, or an error message if the operation fails.

**Developer Notes**
------------------

### Required Fields

*   `settings`

### Request/Response Structure

The request body should be in JSON format and contain the following properties:

```json
{
  "settings": {
    "Challenges": [
      "string_value"
    ],
    "AdditionalData": {},
    "Uuid": "string_value",
    "SingleChallengeMechanisms": "string_value",
    "DurationInMinutes": 0,
    "Name": "string_value"
  }
}
```

The response body will contain an authentication profile object with the same properties as the request body, or an error message if the operation fails.

### Error Handling

If the operation fails, the endpoint will return a 200 status code with an error message in the response body.

### Security Considerations

This endpoint requires bearer authentication. The `x-idap-anon` metadata indicates that anonymous access is not allowed.
