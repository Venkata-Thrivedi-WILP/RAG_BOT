**Important Details for Developers**
--------------------------------------

* The `x-idap-anon` header is set to `False`, indicating that anonymous access is not allowed.
* The request body must be in JSON format with a single property `path`.
* The `Content-Type` header must be set to `application/json`.
* The response schema includes a `Result` field indicating whether the policy block was successfully deleted, and an `Error` field containing error message text on failure.

**Original Documentation**
-------------------------

### Endpoint: Delete Policy

#### Path: `/Policy/DeletePolicyBlock`

#### Method: `POST`

#### Tags: `Policy Management`

#### Metadata:

| Key | Value |
| --- | --- |
| `x-idap-anon` | `False` |
| `x-codegen-request-body-name` | `payload` |

#### Request Body

* Required
* `Content-Type`: `application/json`
* Schema Properties:
	+ `path` (string): The policy block to delete.
	+ Required fields: `path`

#### Sample Request JSON:

```json
{
  "path": "string_value"
}
```

#### Responses:

| Status Code | Description | Content Type | Response Schema |
| --- | --- | --- | --- |
| 200 | API-Result | */* | PolicyDeletePolicyBlock |
|  |  |  | `Result` (boolean): Did the policy block delete succeed. |
|  |  |  | `Error` (object): Error message text on failure, may be null |

#### Security:

* bearerAuth
